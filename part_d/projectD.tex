\section*{Aufgabe 1}
\subsection*{Enviroment Setup}
\begin{enumerate}
 \item Stellt sicher das Python in mindestens version 3.4 installiert ist

\item Stellt sicher das "pip" installiert ist:\\
Ubuntu/Linux Mint:
\begin{lstlisting}
sudo apt-get install python3-pip
\end{lstlisting}
Arch Linux:
\begin{lstlisting}
sudo pacman -S python3-pip
\end{lstlisting}

\item Installiert virtualenv
\begin{lstlisting}
sudo pip install virtualenv
\end{lstlisting}
ODER:
\begin{lstlisting}
sudo pip3 install virtualenv
\end{lstlisting}

\item cd in das Projektverzeichnis
\begin{lstlisting}
cd /Pfad/zum/SE/Projekt
\end{lstlisting}

\item Aktiviert die virtuelle Umgebung mit der Django Installation. (Möglicherweise muss heir die VIRTUAL_ENV Variable angepasst werden).
\begin{lstlisting}
source bin/activate
\end{lstlisting}

\item Zum aktivieren des Entwicklungsservers in das PROJECT/pizza Verzeichnis gehen und mittels
\begin{lstlisting}
python manage.py runserver
\end{lstlisting}
den Server starten. (Erreichbar unter 127.0.0.1:8000)

\item Deaktiviert die virtuelle Umgebung mit:
\begin{lstlisting}
deactivate
\end{lstlisting}
\end{enumerate}

\subsection*{Nutzung (Kunde/Angestellter/Admin)}

In die Kundensicht gelangt man über die Adresse: \url{http://127.0.0.1:8000/} von woaus die Bedinung intuitiv sein sollte.\\ \\

In die Angestelltensicht gelangt man über die Adresse: \url{http://127.0.0.1:8000/staff/open_orders}, welche noch intuitiver ist und zur Zeit nicht Passwort geschützt ist.\\ \\

Da Pizzen statisch sein sollen kann der Angesetllte die Rezepte nicht ändern. Dies ist jedoch dem Admin über die Adresse \url{http://127.0.0.1:8000/admin/} erlaubt. Da dies jedoch nicht mehr Teil der Aufgabenstellung ist bleibt es dem Leser selber überlassen das nötige Passwort zu finden ;-).
